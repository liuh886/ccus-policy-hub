# Governance Configuration and Generated Reports

This directory contains quantitative governance configuration and generated audit outputs. It is not a second documentation authority; the active operating documents are in the parent directory.

## Active configuration

- `audit_thresholds.json` defines quantitative minimums used by the audit pipeline.

Changes to thresholds alter release behavior and require an explanation, before/after results, tests where applicable, and human approval.

## Generated reports

- `reports/` contains audit, statistics, coordinate-governance, or quality outputs generated by maintenance commands.
- Generated reports may be ignored by Git and should be regenerated from the current database rather than treated as permanent source documents.

## Source assets

Large source workbooks may be provided locally and may not exist in a fresh clone. The current facility import script defaults to `agent/ccus-ai-agent/assets/IEA CCUS Projects Database 2026.xlsx`, but an agent must verify the actual supplied file, version, sheet, field mapping, and permission to use it before import.

## Archive

Historical governance documents and task records are reference material only. They must not override `AGENTS.md`, `SKILL.md`, `ARCHITECTURE.md`, `METHODOLOGY.md`, `RUNBOOK.md`, or `SAFETY.md`.

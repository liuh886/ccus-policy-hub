# Skill: FSRTM Framework Evolution & Auditability

本技能定义了 FSRTM 评估标准的迭代机制，以及如何确保底层数据（URL/原文）足以支撑跨版本的重新评分。

## 1. 体系版本控制 (Framework Versioning)

*   **当前版本**：FSRTM v5.0 (100分制)
*   **版本映射**：每条政策在数据库中必须记录 `frameworkVersion`，标记它是基于哪一套标准打的分。
*   **演进触发**：当全球出现新的治理标杆（如国际碳信用新协议）时，需更新本技能的评分锚点。

## 2. 溯源主权原则 (Source Preservation)

为了支撑未来的重新打分，**“证据链的持久性”**高于一切：

### 2.1 URL 强制固化
*   **规则**：必须提供政策原文的永久链接（如官方公报、政府内阁网站）。
*   **失效处理**：如果原始 URL 失效且无备份，该政策的治理分值在下一次体系升级时将强制降级为 `Pending Re-verify`。

### 2.2 文本存档要求
*   **核心逻辑**：解析过程中提取的关键条文（Citation）必须完整记录在 JSON 数据库中，而不仅仅是页码。
*   **多格式兼容**：支持从 `pdf`, `docx`, `html` 中提取。如果解析不到位，必须保留原始文档的路径/哈希值。

---

## 3. 批量重估工作流 (Batch Re-scoring Workflow)

当 FSRTM 标准从 v5.0 升级到 v6.0 时：

1.  **差异分析**：明确新旧版本在哪个维度（如 M 维度）的定义发生了变化。
2.  **源头追溯**：通过脚本提取所有关联政策的 `url` 和 `citation`。
3.  **重新解析**：审计员（或 AI 代理）基于原文进行二次阅读，根据新标准给出新分值。
4.  **影子更新**：在 `auditNote` 中记录：`System Upgrade: Re-scored from v5.0 to v6.0 based on new cross-border requirements.`

---

## 4. 体系健康度指标

一个健康的治理体系应满足：
*   **原文覆盖率**：100% 的政策具备有效 URL。
*   **证据覆盖率**：>80% 的高分维度具备 `citation` 原文引用。
*   **版本一致性**：全库 90% 以上的政策采用最新的 `frameworkVersion`。
